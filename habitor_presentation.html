Machine Learning Terms Simplified

Before we dive into the details, let's understand the key concepts we'll be using

What is Machine Learning?

Machine learning is like teaching a computer to learn patterns from examples, rather than programming it with specific rules. It's similar to how humans learn from experience.

What is a Neural Network?

A neural network is inspired by how our brain works. It contains layers of connected "neurons" that process information. Each connection has a "weight" that changes as the network learns.

What is Gradient Boosting?

Gradient Boosting is like having a team where each new member focuses on fixing the mistakes of the entire team so far. It builds a strong model by combining many simple models.

What is an Ensemble Model?

An ensemble model combines multiple models to make better predictions than any single model alone - like asking for opinions from different experts and then making a decision.

What is BI-LSTM?

Bidirectional Long Short-Term Memory

BI-LSTM is a specialized type of neural network designed to remember patterns over time and process sequences in both directions.

  • Bidirectional: Processes data from both past-to-future and future-to-past directions
  • Long Short-Term Memory: Can remember important information and forget irrelevant details
  • Advanced Pattern Recognition: Excellent at finding hidden patterns in sequential data
  • Context Understanding: Better understands the meaning of data by seeing the complete context

Why BI-LSTM for Student Habits?

Student habits form patterns over time. BI-LSTM excels at analyzing these temporal patterns from both directions to make more accurate predictions about academic performance.

Understanding Model Evaluation Metrics

Accuracy

How often the model's predictions are correct. Like a student's overall score on a test.

Example: 92% accuracy means the model correctly predicts student grades 92 out of 100 times.

Mean Absolute Error (MAE)

The average difference between predicted grades and actual grades, ignoring whether predictions are too high or too low.

Example: MAE of 0.15 means predictions are off by 0.15 GPA points on average.

R² Score (R-squared)

How much of the grade variation is explained by the model, from 0% to 100%.

Example: R² of 0.89 means 89% of the differences in student grades can be explained by the habits we measured.

F1-Score

A balance between precision (correctly identified high-performers) and recall (finding all actual high-performers).

Example: F1-Score of 0.91 means the model is very good at both tasks.

Habitor Project Presentation

Additional Evaluation Metrics

Root Mean Square Error

Similar to MAE, but gives more weight to large errors. It punishes big mistakes more severely.

Example: RMSE of 0.19 means the typical error is about 0.19 GPA points.

Confusion Matrix

A table showing correct and incorrect predictions. It helps us understand what types of mistakes the model makes.

Example: A perfect model would only have numbers along the diagonal.

Which Metrics Matter Most?

  • For General Performance: Accuracy and R² Score
  • For Error Magnitude: MAE and RMSE
  • For Class Imbalance: F1-Score
  • For Error Types: Confusion Matrix

For our student grade predictions, we focus on accuracy, MAE, and R² to provide a complete performance picture.

BI-LSTM Model Details

  • Bidirectional Processing: Captures dependencies in both past and future context
  • Input Layer: 45 engineered features in sequential form
  • Embedding Layer: 64-dimensional space for categorical features
  • BI-LSTM Layer: 128 units in each direction (forward & backward)
  • Dropout Layer: 30% dropout rate to prevent overfitting
  • Dense Layers: 128 units with ReLU activation
  • Output Layer: Sigmoid activation for regression (CGPA prediction)
import tensorflow as tf
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense, LSTM, Dropout, Embedding, Bidirectional

def create_bilstm_model(input_shape, num_categorical_features):
    model = Sequential()
    
    # Embedding layer for categorical features
    model.add(Embedding(input_dim=num_categorical_features + 1, 
                       output_dim=64, 
                       input_length=input_shape[1]))
    
    # Bidirectional LSTM layer
    model.add(Bidirectional(LSTM(128, return_sequences=False)))
    
    # Dropout for regularization
    model.add(Dropout(0.3))
    
    # Dense hidden layer
    model.add(Dense(128, activation='relu'))
    
    # Output layer
    model.add(Dense(1, activation='sigmoid'))
    
    # Compile model
    model.compile(
        optimizer=tf.keras.optimizers.Adam(0.001),
        loss='mean_squared_error',
        metrics=['mae', 'mse']
    )
    
    return model

Ensemble Approach

  • Combined Strength: Leverages complementary abilities of both models
  • Independent Training: Each model trained separately on same data
  • Weighted Averaging: BI-LSTM (60%), Gradient Boosting (40%)
  • Confidence Metrics: Each prediction includes reliability scores
  • Model Selection Logic: Automatically selects more reliable model for certain patterns
  • Benefits:
    • Improved accuracy (+5% over individual models)
    • More consistent predictions
    • Balances complexity with interpretability
def ensemble_predict(bilstm_model, gb_model, X, bilstm_weight=0.6):
    # Get predictions from both models
    bilstm_pred = bilstm_model.predict(X)
    gb_pred = gb_model.predict(X)
    
    # Calculate weighted average
    gb_weight = 1 - bilstm_weight
    ensemble_pred = (bilstm_weight * bilstm_pred + 
                     gb_weight * gb_pred.reshape(-1, 1))

        if abs(bilstm_conf[i] - gb_conf[i]) > 0.3:
